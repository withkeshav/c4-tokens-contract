Technical Verification Notes

  1. Shared Address Mechanism: The contract addresses on Arbitrum (c4 AUD), Base (c4 CAD), Avalanche (c4 CHF), Optimism (c4 SGD), and Polygon (c4 USD) are identical (0x831490d955a5168f44d104173bfd72573b92efea) as they were deployed from the same creator key using identical transactional parameters. Similarly, Ethereum (c4 EUR) and BNB Chain (c4 GBP) share the address 0x30d68bf1ee9e26950dfb9a3db5011a869c570836.
  2. Dynamic Total Supplies: Every deployed token contains a public mint method restricted to the contract owner. Consequently, supplies will shift over time during forensics training courses. For instance, c4 USD (Polygon) now stands at 1.1 trillion due to secondary issuance.
  3. Upgradeable Contract Design: The c4 LEARN and c4 JPY tokens utilize the EIP-1967 Upgradeability Proxy pattern. c4 LEARN points to the implementation at 0x30d68bf1ee9e26950dfb9a3db5011a869c570836, and c4 JPY points to the implementation at 0x8a2d771cc2725b68e5007fcedd0aea6b848e35eb.
  4. Non-Standard Decimals: Unlike conventional ERC-20s that default to 18 decimals, c4 JPY utilizes 2 decimals, c4 LEARN uses 4 decimals, and the TRON-based c4 CNY uses 6 decimals.
